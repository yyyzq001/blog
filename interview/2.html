    # 2.解释下BFC，以及在日常项目中如何使用BFC

    ​ BFC指块级格式化上下文，本质上是为了解决浮动的元素和标准流的元素产生的各种问题，块级格式化让整个页面分成了一个个独立的块级格式，使每个块级格式中的内容元素随便定位，清浮动而保证他们互不影响。比如，一个页面分为2个div块元素，上面的元素设置为浮动流，下面的标准流元素就会占据上面元素的位置，这个时候给下面元素添加属性overfloat：hidden；生成BFC，就会让上面的元素的位置重新出现。同理，还有解决两个元素外边距位置重合导致外边距塌陷的问题也能够用生成BFC来解决。

    # 3.[]==![]返回true还是false，为什么？

    ​ 首先说答案true，

    ​ ==为js中的相等符号，相等符号有一个特点，就是等号两边的操作数类型不一样的情况下，会先转换为同一类型再开始比较（一般称为强制转换）。转换的规律有一个内在逻辑和顺序：

    ​ 1.如果有一个操作数是布尔值，则在比较相等性之前先将其转换为数值----false转换为0，true转换为1；

    ​ 2.如果一个操作数为字符串，一个为数值，则会先将字符串转换为数值；

    ​ 3.如果一个操作数是对象，另一个不是，则调用对象的valueOf()方法，得到的基本类型值按照前面的规则进行比较,如果对象没有valueOf（）方法，则使用toString（）方法。

    ​ 比较时也有几个规则需要遵循：

    ​ 1.null和undefined是相等的；

    ​ 2.要比较相等性之前，不能将null和undefined转换为其他任何值；

    ​ 3.如果有一个操作数为NaN，则相等操作符返回false，而不相等操作符返回true。重要提示：即使两个操作数都是NaN，也会返回false，因为按照规则，NaN不等于NaN；

    ​ 4.如果两个操作数都是对象，则比较他们是不是同一对象，如果两个操作数都指向同一对象，则相等操作符返回true，否则返回false。

    ​ 额外要注意一点，[],{}都属于引用类型，引用类型是存放在堆内存中的，而栈内存中会有一个或多个地址来指向这个堆内存相对应的数据。所以在使用==操作符的时候，对于引用类型的数据，比较的是地址，而不是真实的值。所以[]==[]得到的是false。

    ​ 回到正题，[]==![]为什么为true：

    ​ 1.按照规则首先处理的是：！运算符的优先级是大于==运算符的，所以先看！[],！运算符能将变量转换为boolean类型，且规定，null,undefined,NaN和“ ”空字符串取反都为true，其余都为false，即！[]为false。

    ​ 2.处理[]==false,此时操作数里面出现了一个boolean值，所以先转换boolean值为数值，即false为0。

    ​ 3.处理[]==0,操作数里面一边为对象，一边为数值，先处理对象，空数组按规则调用toString（）方法为空字符串“ ”。

    ​ 4.处理“ ”==0，一个为数值，一个为字符串的情况下，字符串先转换为数值，即“ ”为0.

    ​ 5.至此，得到0==0这一结果，所以返回的就是true了。

    ​ {}==!{}为false也是同理，只是在{}.toString()返回的NaN，按照规则，只要有NaN，则返回false。

    # 4.写出数组方法map，foreach，push，pop，shift，unshift各返回什么

    ​ map()返回一个新的数组，数组的元素为原始数组元素函数处理之后的值。

    ​ Foreach()返回当前数组，元素看处理的方法。

    ​ Push()返回当前数组，在尾部添加元素。

    ​ Pop()返回当前数组，在尾部删除元素。

    ​ unshift()返回当前数组，在头部添加元素。

    ​ shift()返回当前数组，在头部删除元素。

    # 6.如何实现ES5里面的继承（写出你认为最优的继承方式）

    寄生组合式继承：通过借用构造函数来继承属性，通过原型链的混成形式来继承方法

    该方法将原型链继承，构造函数继承，原型式继承的优点进行了一次整合，也是ES5里面最为成熟的继承方式。

    function Parent(name) {
		this.name = [name]
}

Parent.prototype.getName = function() {
		return this.name
}

function Child() {
		//构造函数继承
		Parent.call(this,"zhangsan")
}

//原型链继承
// Child.prototype =new Parent()
//原型式继承，相比于寄生式继承多了一次浅拷贝操作，保证子类原型不会改变父类原型的方法
Child.prototype = Object.create(Parent.prototype)
//增强对象
Child.prototype.constructor = Child

//测试
const child =  new Child()
const parent = new Parent()

child.getName()     //["zhangsan"]
parent.getName()    //undefined

    # 7.vue中v-if和v-for为什么不能一起用

    ​ 因为v-if和v-for同时使用时再vue里面会有一个优先级关系，即v-for优先于v-if，所以每次都是先执行v-for再执行v-if，无论v-if作用于哪几条数据，每次v-for执行时所有元素都要执行一次v-if这是对性能和算力的极大浪费，所以最好的选择是将v-for的对象作为计算属性使用computed以达到v-for的效果。

    # 8.vue中v-model原理是什么

    ​

    v-model是一个实际上给dom元素添加了，绑定数据v-bind和v-on:input监听事件两个方法的语法糖

    <input v-model="searchText">

    等价于：

    <input
	v-bind:value="searchText"
	v-on:input="searchText = $event.target.value"
>

    # 9.vue中双向绑定原理

    vue数据双向绑定是通过数据劫持结合发布者-订阅者模式的方式来实现的。那么数据劫持的具体实现则是通过Object.defineProperty()来实现数据劫持的。这个方法主要是控制了对象属性的一些操作，比如读写权，修改对象属性的get和set方法。双向绑定就是通过这个基础，建立了一套发布者-订阅者模式，用来实现整个流程。

    该模式首先要实现：

    一个监听者Observer，用来劫持并监听所有属性，只要数据发生变动，就会通知对应订阅者。

    一个订阅者Watcher，可以有多个，主要是用来收到属性的变化通知并执行相应的函数，从而更新视图。

    一个解析器compile，可以扫描和解析每个节点的相关指令，并更具初始化模板数据以及初始化相应的订阅器。

    # 10.写出你用过的webpack loader 和plugin

    详细文章

    常用loader

    样式loader：style-loader , css-loader , less-loader , scss-loader

    文件loader：raw-loader , file-loader , url-loader

    编译loader：babel-loader , coffee-loader , ts-loader

    校验loader：mocha-loader , jshint-loader , eslint-loader

    常用plugin

    首先webpack内置*UglifyJsPlugin*，压缩和混淆代码。

    webpack内置***CommonsChunkPlugin***，提高打包效率，将第三方库和业务代码分开打包。

    ProvidePlugin：自动加载模块，代替require和import

    new webpack.ProvidePlugin({
      $: 'jquery',
      jQuery: 'jquery'
    })

    ***html-webpack-plugin***可以根据模板自动生成html代码，并自动引用css和js文件

    *extract-text-webpack-plugin* 将js文件中引用的样式单独抽离成css文件

    *DefinePlugin* 编译时配置全局变量，这对开发模式和发布模式的构建允许不同的行为非常有用。

    new webpack.DefinePlugin({
    PRODUCTION: JSON.stringify(true),
    VERSION: JSON.stringify("5fa3b9"),
    BROWSER_SUPPORTS_HTML5: true,
    TWO: "1+1",
    "typeof window": JSON.stringify("object")
  })

    HotModuleReplacementPlugin 热更新

    1. 添加HotModuleReplacementPlugin
    2. entry中添加 "webpack-dev-server/client?http://localhost:8080/ (opens new window)",
    3. entry中添加 "webpack/hot/dev-server"

    (热更新还可以直接用webpack_dev_server --hot --inline,原理也是在entry中添加了上述代码)

    webpack 内置的***DllPlugin***和***DllReferencePlugin***相互配合，前置第三方包的构建，只构建业务代码，同时能解决Externals多次引用问题。DllReferencePlugin引用DllPlugin配置生成的manifest.json文件，manifest.json包含了依赖模块和module id的映射关系

    babili-webpack-plugin、transform-runtime 、transform-object-rest-spread

    1. babili-webpack-plugin:构建在babel之上，它的用处可以看BabiliWebpackPlugin (opens new window)
    2. transform-runtime :解决了babel在每个文件都插入了辅助代码，代码体积过大的问题。
    3. transform-object-rest-spread：Transform rest properties for object destructuring assignment and spread properties for object literals，为对象字面量添加解构赋值和spread属性

    *optimize-css-assets-webpack-plugin* 不同组件中重复的css可以快速去重

    webpack-bundle-analyzer 一个webpack的bundle文件分析工具，将bundle文件以可交互缩放的treemap的形式展示。

    compression-webpack-plugin 生产环境可采用gzip压缩JS和CSS

    happypack：通过多进程模型，来加速代码构建

    const os = require('os');
      let HappyPack = require('happypack');
      let happyThreadPool = HappyPack.ThreadPool({size: os.cpus().length});
      exports.plugins = [
        new HappyPack({
          id: 'jsx',
          threadPool: happyThreadPool,
          loaders: [ 'babel-loader' ]
        }),

        new HappyPack({
          id: 'coffeescripts',
          threadPool: happyThreadPool,
          loaders: [ 'coffee-loader' ]
        })
      ];

      exports.module.loaders = [
        {
          test: /\.js$/,
          loaders: [ 'happypack/loader?id=jsx' ]
        },
        {
          test: /\.coffee$/,
          loaders: [ 'happypack/loader?id=coffeescripts' ]
        },
      ]

    # 11.前端如何做优化（如何让网页打开的更快）

    一、content方面：

    ​ 1.减少http请求：合并文件，压缩文件，css精灵图，inline image

    ​ 2.减少DNS查询：DNS查询完成之前浏览器无法从这个主机下载任何文件。

    ​ 方法：DNS缓存，将资源分布到恰当数量的主机名，平衡并行下载和DNS查询。

    ​ 3.避免重定向：多余的中间访问。

    ​ 4.使Ajax可缓存。

    ​ 5.非必须组件延迟加载：懒加载。

    ​ 6.未来所需组件预加载。

    ​ 7.减少dom元素数量。

    ​ 8.将资源放到不同的域下：浏览器同时重一个域下载资源的数目有限，增加域可以提高并行下载量。

    ​ 9.图片懒加载。

    ​ 10.减少iframe。

    ​ 11.不要404。

    二、Server方面

    1.使用CDN

    2.添加Expires或者Cache-Control响应头

    3.对组件使用Gzip压缩

    4.配置ETag

    5.Flush Buffer Early

    6.Ajax使用GET进行请求

    7.避免空src的img标签

    三、cookie方面

    1.减小cookie大小

    2.引入资源的域名不要包含cookie

    四、CSS方面

    1.将样式表放到页面顶部

    2.不使用CSS表达式

    3.尽量不使用@import

    4.不使用IE的Filter

    五、JavaScript方面

    1.将脚本放到页面底部

    2.将javascript和css从外部引入

    3.压缩javascript和css

    4.删除不需要的脚本

    5.减少DOM访问

    6.合理设计事件监听器

    六、图片方面

    1.优化图片：根据实际颜色需要选择色深、压缩

    2.优化css精灵

    3.不要在HTML中拉伸图片

    4.保证favicon.ico小并且可缓存

    七、移动方面

    1.保证组件小于25k

    2.Pack Components into a Multipart Document
